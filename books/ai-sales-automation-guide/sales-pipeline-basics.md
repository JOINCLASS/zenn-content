## 営業パイプラインとは何か

営業パイプラインとは、見込み客との最初の接点から成約に至るまでの流れを、段階（ステージ）に分けて管理する仕組みです。

一般的なステージ構成はこのようになります。

```
リード → 初回連絡 → 商談中 → 提案・見積 → 交渉 → 受注 / 失注
```

各ステージにいま何件の案件があるかを把握することで、以下が見えるようになります。

- 来月の売上見込みはどのくらいか
- どの案件にフォローが必要か
- 営業活動のボトルネックはどこか

これが「商談の見える化」の基本です。大企業ではSalesforceやHubSpotで当たり前に行われていますが、フリーランスや少人数チームではまだExcelやスプレッドシートで管理しているケースが多いのが実情です。

本章では、パイプライン管理の基本概念から、Excel管理の限界、そしてカンバンボードへの移行までを解説します。

## なぜパイプライン管理が必要なのか

「案件が3件しかないなら頭で管理できる」。確かにそうかもしれません。しかし、営業を続けていると案件数は増えていきます。そして案件管理が破綻するのは、案件が「増えた瞬間」ではなく、「増え始めた時期に管理の仕組みを持っていなかった」場合に起きます。

パイプライン管理がもたらす3つの価値を整理します。

### 1. 売上予測の精度が上がる

各ステージの案件数と、ステージごとの平均成約率がわかれば、将来の売上をある程度予測できます。

```
例：
商談中 5件 × 平均成約率30% × 平均単価50万円 = 75万円（見込み売上）
提案済 3件 × 平均成約率60% × 平均単価50万円 = 90万円（見込み売上）
合計の期待売上 = 165万円
```

この計算が頭の中にあるだけで、「今月は新規リードの開拓を強化すべきか、既存案件の成約を優先すべきか」の判断ができるようになります。

### 2. ボトルネックが可視化される

パイプラインを可視化すると、特定のステージに案件が滞留していることに気づけます。

たとえば「商談中」に案件が集中しているなら、提案書の作成が追いついていない可能性があります。「提案済み」に溜まっているなら、相手のレスポンスが遅いのか、提案内容が刺さっていないのかを検討すべきです。

ボトルネックが見えれば、打ち手が見えます。

### 3. フォロー漏れを防げる

案件が5件を超えると、「あの会社にフォローメールを送ったか？」「見積もりはいつ出したっけ？」と記憶が曖昧になります。パイプラインを管理していれば、各案件の現在の状態と最終アクション日が一目で分かるので、フォロー漏れを構造的に防げます。

## Excel管理の限界

Excel（Googleスプレッドシート含む）で商談管理をしている方は多いと思います。最初のうちは問題なく回ります。しかし、案件が10件を超えたあたりから、以下の問題が表面化します。

### ステータスの更新を忘れる

商談が進んでもシートを更新するのを忘れ、実際の状況と管理表がずれる。週末にまとめて更新しようとして、どこまで進んでいたか思い出せない。

「Excel管理は、更新してこそ意味がある」のに、更新する手間が原因で使わなくなる。このパラドックスに多くの人がハマります。

### 全体像が見えない

行が増えるほどスクロールが必要になり、「いま提案中の案件が何件あるか」を把握するのにフィルタやピボットを駆使する必要がある。一目で分かる状態にはなりません。

Excel はデータを「表」として管理するのが得意ですが、営業のステージを「流れ」として可視化するのには向いていません。

### 手入力のコストが高い

新しい問い合わせが来るたびに、メールの内容を読み取って会社名・担当者名・連絡先をセルに手入力する。1件15分の作業でも、月に15件あれば約4時間です。

この「最初の入力」が面倒で、管理表への記録が後回しになる。後回しが積み重なると、管理表自体が信用できない状態になる。こうしてExcel管理は形骸化していきます。

### チーム共有がしづらい

複数人で同じシートを編集すると、上書き事故や「誰がどこを更新したか分からない」問題が起きます。コメントやチャットで補足が必要になり、情報が分散します。

つまりExcelは「記録する場所」にはなっても、「営業活動を前に進めるツール」にはなりにくいのです。

## カンバンボードの威力

パイプライン管理で最も直感的なのが、カンバンボード形式です。

```
| リード | 商談中 | 提案済み | 受注 |
|--------|--------|----------|------|
| A社    | C社    | E社      | G社  |
| B社    | D社    |          |      |
|        |        |          |      |
```

縦にカード（案件）、横にステージ。この配置だけで、いま営業活動がどういう状態にあるかが一目で分かります。

カンバンの利点は3つあります。

### 1. 視覚的に把握できる

どのステージに案件が集中しているか、滞留しているカードはないか、が瞬時に分かります。「提案済みのカードが5枚溜まっている。先方の返答待ちが多い」といった気づきが自然に得られます。

### 2. 操作が簡単

案件が次のステージに進んだら、カードをドラッグ&ドロップするだけ。Excelのセルを探して値を書き換える手間がありません。

### 3. チームで共有しやすい

全員が同じボードを見ているので、「あの案件いまどうなってる？」という確認が不要になります。情報の非対称性が解消されます。

## パイプラインのステージ設計

カンバンボードを導入する際に最初に決めるべきは、ステージの構成です。

ステージが多すぎると管理が煩雑になり、少なすぎると粒度が粗くて使い物になりません。フリーランスや少人数チームの場合、3〜5ステージで十分です。

### おすすめのステージ構成

**シンプル版（3ステージ）**
```
リード → 商談中 → 受注/失注
```

**標準版（5ステージ）**
```
リード → 初回連絡済 → 商談中 → 提案済み → 受注/失注
```

**詳細版（7ステージ）**
```
リード → 初回連絡済 → 商談調整中 → 商談完了 → 提案済み → 交渉中 → 受注/失注
```

最初はシンプル版から始めて、運用しながら必要に応じてステージを増やすのが失敗しないコツです。

### ステージ設計のポイント

- **各ステージで「次に何をすべきか」が明確であること**。「商談中」は曖昧ですが、「日程調整中」「提案書作成中」なら次のアクションが分かります
- **ステージ間の移動条件を決めておくこと**。「メールを送ったら初回連絡済み」「見積書を送付したら提案済み」など
- **失注も必ずステージとして設けること**。失注案件を管理することで、「なぜ失注したか」の分析が可能になります

## 自動化で「入力ゼロ」を実現する

カンバンボードで管理すれば視認性は解決します。しかし、もう一つの問題が残っています。それは「最初の入力」です。

新しいリードが来るたびに、メールを読んで案件カードを手動で作成する。これが面倒で続かない、というのがパイプライン管理が定着しない最大の理由です。

ここでAIによる自動化が効きます。次章以降で詳しく解説しますが、メールを転送するだけでAIが案件情報を自動抽出し、カンバンボードに登録する仕組みを使えば、「最初の入力」の壁を完全に取り除けます。

## まとめ

営業パイプライン管理は、大企業だけのものではありません。フリーランスや少人数チームこそ、限られた時間とリソースの中で商談の取りこぼしを防ぐために、パイプラインの可視化が重要です。

本章のポイントを整理します。

- パイプラインは「売上予測」「ボトルネック発見」「フォロー漏れ防止」の3つの価値をもたらす
- Excelは「記録の場所」にはなるが「営業を前に進めるツール」にはなりにくい
- カンバンボードなら、案件の状態が一目で分かり、操作も簡単
- ステージは3〜5段階から始めて、運用しながら調整するのがベスト
- 「最初の入力」のコストが定着の最大の壁であり、AI自動化で解決できる

次章では、フリーランスや個人事業主が営業事務にどれだけの時間を費やしているかを具体的に分析し、その80%がAIで自動化できることを示します。

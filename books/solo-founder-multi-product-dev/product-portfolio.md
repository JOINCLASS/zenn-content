## ShareToku — サブスク紹介プログラム横断検索

### 概要

ShareTokuは、各種サブスクリプションサービスの紹介プログラム（リファラルプログラム）を横断的に検索・比較できるWebサービスです。「友達紹介で割引」というキャンペーンは多くのサブスクで存在しますが、その情報は各サービスのサイトに散在しています。ShareTokuはこれらの情報を一箇所に集約し、ユーザーが最もお得な紹介プログラムを見つけられるようにします。

### 技術スタック

- **フロントエンド/バックエンド:** Next.js 16（App Router + Server Components）
- **データベース/認証:** Supabase（PostgreSQL + Auth）
- **UI:** Radix UI + Tailwind CSS 4
- **データ可視化:** Recharts
- **デプロイ:** Vercel

### ターゲットユーザー

節約志向の消費者、サブスクリプションサービスを多く利用しているユーザー。

## MochiQ — AI学習アプリ

### 概要

MochiQは、AIを活用した学習クイズアプリです。ユーザーが学びたいテーマを入力すると、Gemini AIが最適なクイズを自動生成します。スペースドリピテション（間隔反復）アルゴリズムにより、記憶の定着を科学的に最適化します。カメラで教科書やノートを撮影してクイズを生成する機能、クイズのディープリンク共有機能も備えています。

### 技術スタック

- **モバイルアプリ:** Flutter 3.x + Dart 3.7
- **バックエンド:** Firebase（Auth, Firestore, Analytics）
- **AI:** Gemini AI（google_generative_ai）
- **状態管理:** Provider
- **課金:** in_app_purchase（iOS/Android サブスクリプション）
- **その他:** image_picker（カメラ）、flutter_local_notifications（復習リマインダー）、app_links（ディープリンク）
- **LP:** Next.js + お名前.com（FTPデプロイ）

### ターゲットユーザー

大学受験生、資格試験の学習者、語学学習者。

## Focalize — 営業自動化SaaS

### 概要

Focalizeは、フリーランスや小規模事業者向けの営業自動化SaaSです。Gmail連携によるメール送受信の自動追跡、Googleカレンダー連携による日程調整自動化、商談パイプラインの可視化を提供します。Gemini AIによるメール解析で、フォローアップのタイミングを自動提案する機能も搭載しています。Google CASA Tier 2認証を取得しており、セキュアなGoogleアカウント連携を実現しています。

### 技術スタック

- **フロントエンド/バックエンド:** Next.js 14（App Router + Server Components）
- **データベース/認証:** Supabase（PostgreSQL + Auth）+ pg（直接クエリ）
- **決済:** Stripe
- **国際化:** next-intl（日英対応）
- **認証:** next-auth
- **Google連携:** Google APIs（Gmail, Calendar）
- **通知:** nodemailer + web-push
- **AI:** Gemini AI（@google/genai）
- **UI:** Radix UI + dnd-kit（ドラッグ&ドロップ）+ Tailwind CSS 4
- **デプロイ:** Vercel（output: 'standalone'）

### ターゲットユーザー

フリーランス、小規模事業者、1人社長。営業活動に時間を取られたくないが、商談管理は必要な人。

## 元気ボタン — 高齢者見守りアプリ

### 概要

元気ボタンは、離れて暮らす親御さんの安否を確認するための見守りアプリです。仕組みは究極的にシンプルで、親御さんが毎日ボタンを1つ押すだけ。押されたら子ども側にプッシュ通知が届きます。QRコードによるかんたん初期設定、ホーム画面ウィジェット、リマインダー通知、チェックイン履歴のダッシュボードを備えています。

「監視」ではなく「つながり」をコンセプトに、親御さん側は完全無料で利用できます。

### 技術スタック

- **モバイルアプリ:** Flutter 3.38 + Dart 3.10
- **バックエンド:** Firebase（Auth, Firestore, Cloud Messaging, Analytics）
- **状態管理:** Riverpod + freezed
- **ナビゲーション:** go_router
- **QRコード:** mobile_scanner + qr_flutter
- **通知:** flutter_local_notifications + Firebase Cloud Messaging
- **デプロイ:** App Store / Google Play（Fastlane設定済み）

### ターゲットユーザー

離れて暮らす親を持つ30代〜60代の子ども世代。ITリテラシーが高くない高齢の親御さんでも使えることが必須条件。

## Skill Tracker — スキル管理SaaS

### 概要

Skill Trackerは、チームメンバーのスキルを可視化・管理するSaaSです。Excelでのスキル管理に限界を感じている中小企業やチームリーダー向けに、スキルマトリクスの作成、成長トラッキング、スキルギャップ分析を提供します。現在はLPのみ公開中で、本体は開発中です。

### 技術スタック

- **LP:** Next.js 15（Static Export / `output: "export"`）
- **UI:** Headless UI + Heroicons + Tailwind CSS 4
- **デプロイ:** お名前.com（FTPデプロイ）

### ターゲットユーザー

中小企業の人事担当者、チームリーダー、CTOで、メンバーのスキル管理を体系化したい人。

## プロダクト間のシナジーと独立性

### 技術的なシナジー

5つのプロダクトは独立して動作しますが、技術的な知見は循環しています。

**セキュリティの横展開。** FocalizeでCASA Tier 2認証を取得する過程で実装したセキュリティヘッダー、トークン暗号化、CSPポリシーの設計パターンは、ShareTokuにもそのまま適用しています。1つのプロダクトで苦労して学んだセキュリティ対策が、他のプロダクトでは「最初から正しい実装」としてスタートできる。

**UI/UXの知見共有。** 元気ボタンで培ったシニア向けのアクセシビリティ設計は、他のプロダクトのUX改善にも影響しています。「タップ領域を大きく」「フィードバックを明確に」といった原則は、高齢者向けに限らずすべてのプロダクトで有効です。

**インフラの共通化。** GitHub ActionsのCI/CDパターン、Supabase/Firebaseの設計パターン、Vercelのデプロイ設定は、プロダクト間で大きく変わりません。新規プロダクトの立ち上げが毎回速くなります。

### 意図的な独立性

一方で、プロダクト間の依存関係は意図的にゼロにしています。

- 共有ライブラリやモノレポは使っていない
- データベースはプロダクトごとに完全に分離
- デプロイパイプラインもプロダクトごとに独立
- あるプロダクトに障害が起きても、他のプロダクトには影響しない

この「知見は共有するが、コードとインフラは分離する」という方針が、1人で5プロダクトを運用する上での鍵です。共有コンポーネントを更新したら5プロダクト全部のテストが壊れる、という状況は1人では対処できません。

### ポートフォリオとしてのバランス

5つのプロダクトは、意図的にターゲットと収益モデルを分散させています。

| プロダクト | 収益モデル | 市場 | リスク特性 |
|-----------|----------|------|-----------|
| ShareToku | アフィリエイト/広告 | BtoC | 低リスク・低収益 |
| MochiQ | サブスクリプション | BtoC | 中リスク・中収益 |
| Focalize | SaaS月額課金 | BtoB | 高リスク・高収益 |
| 元気ボタン | フリーミアム | BtoC | 中リスク・社会的価値 |
| Skill Tracker | SaaS月額課金 | BtoB | 開発中 |

BtoBとBtoC、サブスクリプションとアフィリエイト、Webとモバイル。軸を分散させることで、特定の市場やプラットフォームの変動に対するリスクヘッジになります。全てがうまくいく必要はなく、5つのうち1〜2つが軌道に乗れば、全体として持続可能な事業になります。
